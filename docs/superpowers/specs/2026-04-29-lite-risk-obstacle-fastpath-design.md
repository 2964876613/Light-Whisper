# 轻约束风险与障碍输出（快速通道）设计

## 1. 目标与范围

本设计用于在保持“拍一拍”快速返回的前提下，恢复并稳定输出“障碍物信息 + 危险等级”。

核心目标：
- 保持当前快速返回体验，不回退到重 JSON 约束
- 返回结果中包含三类信息：障碍、风险、描述
- 解析逻辑容错，不因格式轻微偏差导致整体失败
- TTS 始终有可播报的画面结论

非目标：
- 不恢复严格结构化 JSON 全字段校验
- 不引入额外多轮状态管理
- 不改变首页交互入口与基础流程

## 2. 问题背景

当前“拍一拍”已切到轻量提示词以降低延迟，返回速度明显提升，但结果主要是自由描述文本，缺少稳定的障碍物提示与风险等级输出。

此前严格 JSON 路径信息更全，但推理负担较高，导致时延明显上升。

因此需要在“速度优先”前提下引入轻约束格式，使障碍与风险可以被稳定提取。

## 3. 方案概览

采用“自然语言 + 轻标签单行”输出协议（非 JSON）：

- 模型目标输出：
  - `障碍:xxx；风险:低|中|高；描述:xxx`
- 说明：
  - 仍是单行文本，token 成本低
  - 标签帮助服务层稳定提取结构信息
  - `描述` 用于 TTS 直接播报

## 4. 服务层设计

### 4.1 轻约束提示词

在 `preferLitePrompt: true` 路径中，提示词改为要求单行轻标签格式，不再要求 JSON：
- 必须包含 `障碍`、`风险`、`描述` 三段
- 风险限定为 `低|中|高`
- 描述尽量短（建议 <= 18 字）
- 不要额外解释文本

### 4.2 轻解析器

在 `lib/services/doubao_api_service.dart` 新增轻解析器（字符串级）：
- 按 `；` 分段
- 识别前缀：`障碍:`、`风险:`、`描述:`
- 容错规则：
  - 兼容中英文冒号
  - 兼容多余空格
  - 段落顺序可不固定
- 解析失败不抛错，只返回可提取到的部分字段

### 4.3 风险兜底推断

当 `风险` 未明确时，本地关键词推断：
- 高：车辆、台阶下沿、坑洼、施工、快速接近
- 中：障碍物、拥挤、昏暗、湿滑
- 低：通畅、无明显障碍

推断只用于补齐风险标签，不覆盖模型原始描述。

### 4.4 结果模型兼容

在现有 `ImageAnalysisResult` 上增加可选轻元信息字段（例如 `liteMeta`）：
- obstacleText
- riskLevel
- briefDescription

若轻标签解析失败，仍保留原 `ttsText` 作为兜底输出，确保每次有可播报结果。

## 5. 页面消费规则

### 5.1 拍一拍结果页（`lib/screens/chat_screen.dart`）

- 主文案：优先显示/播报 `briefDescription`，无则用 `ttsText`
- 补充信息：
  - 障碍：`obstacleText`，缺失显示“未明确”
  - 风险：`riskLevel`，缺失显示“未知”或使用本地推断
- 不因字段缺失阻断页面更新

### 5.2 实时模式（`lib/screens/live_vision_screen.dart`）

- 保持低延迟优先
- TTS 只播报短描述
- 屏幕可选显示障碍/风险补充
- 仍允许字段缺失并平滑降级

## 6. 容错与降级

1. 三字段全有：展示完整结果。
2. 部分字段可解析：展示已有字段，缺失项用“未明确/未知”。
3. 轻标签完全不匹配：回退为纯描述模式，但继续播报。
4. 请求异常：沿用网络兜底文案。

原则：
- 结果不“判死”
- 优先保证有画面结论
- 在可行时补齐障碍与风险

## 7. 性能策略

- 继续使用轻量提示词与小图上传配置
- 不启用重 JSON 解析路径
- 轻解析器采用 O(n) 字符串处理，不引入复杂计算
- 保留现有耗时日志，持续观察请求与总耗时中位数

## 8. 测试与验收

### 8.1 功能验收

- 常规场景：返回包含障碍、风险、描述
- 部分缺失场景：页面仍显示结果并播报
- 无标签场景：仍有可播报描述

### 8.2 性能验收

- 拍一拍首结果时延维持当前快路径水平（不回到重 JSON 慢路径）
- 平均时延不显著劣化

### 8.3 回归验收

- 原有语音播报链路可用
- 实时模式保持可用与低延迟
- 现有网络兜底逻辑不被破坏

## 9. 实施顺序

1. 服务层新增轻标签提示词与轻解析器
2. 在 `ImageAnalysisResult` 中补充轻元信息字段
3. `chat_screen.dart` 接入障碍/风险展示
4. `live_vision_screen.dart` 按需接入轻补充显示
5. 执行功能、性能、回归验证

## 10. 结论

该方案通过“轻标签自然句”替代严格 JSON，在速度优势基本不变的前提下，恢复障碍物和危险等级输出。它符合当前作业场景的优先级：先保证可用与响应，再在轻约束下提升信息完整度。
